Design Overview

The Phones-R-Us project required the development of two precision casings for their modular mobile device. The bottom casing was modelled directly from a detailed client drawing provided by Phones-R-Us, which included all necessary dimensions, tolerances, and geometric specifications. This drawing was clear and well-structured, allowing for straightforward translation into CAD with minimal interpretation required.

The top casing presented a different challenge, as it required complete reverse engineering from a physical sample part. Unlike the bottom casing, no formal drawing existed for the top casing initially. A reference drawing was provided, but it had significant limitations that required careful analysis and problem-solving to overcome. Both components were designed in Autodesk Fusion 360, a professional CAD software that enabled precise modelling of complex geometries, including the curved surfaces and non-uniform depth of the top casing.

The design process involved multiple iterations to ensure proper fit between the two casings, as well as compatibility with existing components from other suppliers. The final designs were exported as STL files, the industry-standard format for additive manufacturing, ready for the 3D printing phase of the project.

2D CAD Drawings

Top Casing Drawing

Dimensions: 114mm × 60mm × 11.7mm
⚠️ Drawing Challenges: The top casing drawing provided by Phones-R-Us had multiple missing dimensions and confusing cross-sections that made accurate interpretation difficult. A key issue was that it was often unclear where exactly on the top casing each measurement applied, as the drawing lacked clear reference points and datum indicators. Some critical dimensions were completely absent from the drawing, while others were placed ambiguously on the drawing sheet with unclear leaders pointing to uncertain locations. The cross-sections provided did not always align with the main views, creating further confusion about which features corresponded to which measurements.
✓ Solution Approach: To resolve these issues, missing values were systematically calculated by cross-referencing other known measurements on the drawing and using geometric relationships between features. Multiple views (front, top, side, and isometric) were compared to determine the correct locations for each ambiguous dimension. Logical assumptions had to be made where information was incomplete, and these assumptions were validated through test prints and fit checks with the bottom casing. This methodical approach resulted in an accurate CAD model despite the drawing's limitations.

Fusion 360 3D Models

Top Casing - 3D Model

The top casing 3D model was built in Fusion 360 using the reverse engineered dimensions derived from the physical sample and the problematic drawing. The top casing features a non-flat geometry, with depth varying across the part rather than being a consistent 11.7mm throughout. This required careful modelling of curved surfaces, transitions between different depth zones, and the integration of snap-fit features for assembly. The model was iteratively refined based on test prints to ensure proper fit with the bottom casing.

STL Exported

Bottom Casing - 3D Model

The bottom casing 3D model was developed directly from the client's engineering drawing. With a consistent depth of 13mm and clear dimensions throughout, the modelling process was efficient and straightforward. Key features modelled included the battery compartment, mounting bosses for the printed circuit board, alignment ribs for the top casing, and the outer profile matching the phone dimensions. The model was verified against the drawing at each stage before STL export.
